# 사용방법

이 프로젝트는 `make` 명령으로 Docker Compose를 실행합니다.
평소에는 긴 `docker compose -f ...` 명령을 직접 입력하지 않아도 됩니다.

## 기본 개념

서비스는 두 그룹으로 나눕니다.

| 그룹 | 포함 서비스 | 설명 |
| --- | --- | --- |
| `infra` | DB, Airflow, FTP | 자주 재빌드하지 않는 기반 서비스 |
| `app` | API, Web, Nginx, MinIO, MinIO init, dev ADFS | 실제 앱 기능 서비스 |

`infra`는 한 번 켜두고, 개발 중에는 보통 `app`만 다시 올리거나 빌드합니다.

## 개발 환경 실행

로컬 dummy ADFS/RAG/메일/Jira를 쓰는 일반 개발 환경입니다.

```bash
make dev
```

위 명령은 아래 순서로 실행됩니다.

```bash
make dev-infra-up
make dev-app-up
```

## 개발 중 자주 쓰는 명령

app만 다시 올리기:

```bash
make dev-app-up
```

app만 다시 빌드하기:

```bash
make dev-app-build
```

app만 내리기:

```bash
make dev-app-down
```

infra만 올리기:

```bash
make dev-infra-up
```

infra만 다시 빌드하기:

```bash
make dev-infra-build
```

infra만 내리기:

```bash
make dev-infra-down
```

## OIDC 개발 환경

실제 OIDC 연동 개발용입니다.

```bash
make oidc
```

OIDC app/infra만 따로 조작할 때:

```bash
make oidc-app-up
make oidc-app-build
make oidc-app-down

make oidc-infra-up
make oidc-infra-build
make oidc-infra-down
```

## 운영 구성

운영 compose 조립 기준입니다.

```bash
make prod
```

운영 app/infra만 따로 조작할 때:

```bash
make prod-app-up
make prod-app-build
make prod-app-down

make prod-infra-up
make prod-infra-build
make prod-infra-down
```

## 전체 종료

dev, OIDC, 운영 compose project를 모두 내립니다.

```bash
make down
```

특정 환경의 app 또는 infra만 내리고 싶으면 `*-app-down`, `*-infra-down`을 사용합니다.

## 검증 명령

개발 API 컨테이너 기준으로 실행합니다.

```bash
make check-api
make test-api
make makemigrations-check
```

## 파일 구조

root에는 사람이 실행하는 compose 파일만 둡니다.

```text
docker-compose.dev.yml
docker-compose.oidc.yml
docker-compose.yml
Makefile
```

내부 조립용 compose 파일은 `compose/` 아래에 있습니다.

```text
compose/airflow.yml
compose/dev.app.yml
compose/dev.infra.yml
compose/oidc.app.yml
compose/oidc.infra.yml
compose/prod.app.yml
compose/prod.infra.yml
```

일반 사용자는 `compose/` 파일을 직접 실행하지 않고 `make` 명령을 사용하면 됩니다.
